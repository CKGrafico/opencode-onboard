# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in opencode-onboard, please report it responsibly.

**Do not open a public GitHub issue for security vulnerabilities.**

Instead, please email the maintainer directly at the address listed on the [GitHub profile](https://github.com/CKGrafico). Include the following information:

- A description of the vulnerability
- Steps to reproduce the issue
- The potential impact
- Any suggested fixes or mitigations

## Response Timeline

- **Acknowledgment**: within 48 hours of receiving the report
- **Initial assessment**: within 7 days
- **Fix or mitigation**: depends on severity and complexity, communicated to the reporter

## Scope

This security policy applies to the `opencode-onboard` npm package and its source code in this repository. It does not apply to third-party tools that opencode-onboard installs or configures (such as OpenCode, OpenSpec, codegraph, or agentmemory). For vulnerabilities in those tools, please contact their respective maintainers.

## Responsible Disclosure

We appreciate responsible disclosure and will credit reporters in the release notes that fixes the vulnerability, unless the reporter prefers to remain anonymous.
